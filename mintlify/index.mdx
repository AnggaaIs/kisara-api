	<Steps>
		<Step title="Create API key from your account">
			Sign in via web session, then generate API key from API key management endpoint or dashboard.
		</Step>
		<Step title="Send requests with API key header">
			For protected endpoints, include `X-API-Key: <your_api_key>`.
		</Step>
		<Step title="Validate the response">
			Use `status_code`, `message`, `data`, and `metadata` to verify and debug responses.
		</Step>
	</Steps>

	<Info>
		JWT bearer token is still used for internal web session flows (login/refresh and API key management), not as primary auth for API consumers.
